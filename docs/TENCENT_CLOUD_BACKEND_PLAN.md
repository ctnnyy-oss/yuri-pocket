# 腾讯云后端方案

这台腾讯云服务器适合做 Yuri Nest 的安全后端，不适合直接跑大模型。

## 最推荐的分工

- GitHub Pages：继续放前端网页，负责手机、平板、电脑访问界面。
- 腾讯云服务器：运行后端 `/api/chat`，保存模型密钥，转发请求到中转站或模型服务。
- 数据库：后续放聊天记录、长期记忆、多设备同步数据；早期可先用 SQLite，稳定后再换 PostgreSQL。

## 为什么需要服务器

公开网页不能内置模型密钥。只要密钥写进前端代码，别人打开浏览器开发者工具就能看到。

服务器可以把密钥放在 `.env` 里，前端只请求服务器，服务器再去调用模型。这样密钥不会进入 GitHub，也不会打包进网页。

## 当前项目已经准备好的部分

- 本地后端入口：`server/index.mjs`
- 聊天接口：`/api/chat`
- 前端可配置云端后端：`VITE_API_BASE_URL`
- 本地开发默认仍然请求 `/api/chat`，由 Vite 转发到 `127.0.0.1:8787`

## 上云前必须补的安全项

- 后端必须走 HTTPS。
- 数据库不能暴露到公网，只允许本机后端访问。
- 模型密钥只放服务器环境变量，不写进前端，不提交 GitHub。
- 后端要加访问控制，避免陌生人消耗妹妹的模型额度。
- 定期备份数据库，尤其是记忆和聊天记录。

## 建议路线

1. 先把腾讯云配置成只跑后端和数据库。
2. 给服务器绑定域名和 HTTPS。
3. 设置 `VITE_API_BASE_URL=https://后端域名` 后重新部署 GitHub Pages。
4. 再做登录、记忆云同步和数据库备份。
