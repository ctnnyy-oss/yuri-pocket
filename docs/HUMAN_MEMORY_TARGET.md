# 人类记忆 80% 代理目标

日期：2026-05-05

## 说明

“接近人类记忆”不能用一句话保证，需要拆成可验证能力。Yuri Nest 先把人类记忆机制翻译成 17 个工程指标，达到 14/17 视为 80%+ 代理达标，达到 16/17 视为 90%+ 代理达标，当前自动门槛已跑到 17/17。

## 本轮新增的“80% 代理”支撑

- 回忆模式：用户问“以前 / 上次 / 还记得 / 记忆 / 档案”时，召回更多长期记忆和旧事件。
- 核心锚点：稳定、置顶、高权重高可信、流程规则不会因为时间久就被轻易压下去。
- 精准捕捉：自动记忆先提取真正要保存的内容，减少口头语污染。
- 长日志：调用日志从 50 条扩到 500 条，保留更长“想起过”的证据。
- 时间证据：进入 prompt 的记忆会带记录、更新、上次调用日期。
- 再巩固提醒：核心记忆太久没调用时，守护台会提示复查确认。
- 语义线索：关键词会扩展为概念线索组，例如记忆/回忆/旧事、架构/模块/重构、文档/图片/PDF 等，不再只靠字面撞词。
- 轻量语义向量：本地把关键词、概念组、中文 2-3 字片段组合成相似度评分，用来处理“怕以后没人维护”这类非原词回忆。
- 重复证据增强：多来源、多版本、重复合并会提升召回评分、可信度和权重，模拟人类记忆的重复巩固。
- 记忆评测集：`npm run test:memory` 会跑旧事召回用例和 8 维代理门槛，旧事召回低于 90% 或代理分低于 80% 会直接失败。
- 反思整理候选：后台整理会把多条同类事件/偏好提炼成 reflection 候选，模拟人类把经历抽象成规则的过程。
- 冲突重巩固：相反偏好会优先标成 value conflict，而不是被普通“相似重复”吞掉；人工编辑会保留版本线。
- 时间线定位：回忆模式会识别“5月5号 / 五一最后一天 / 之前 / 之后”等线索，并把事件时间、来源时间纳入评分。
- 情绪显著性：每条记忆会估算 `emotionalSalience`，用户问“我最焦虑/最在意/放不下什么”时，相关情绪记忆会被优先召回。
- 可解释反思：后台整理现在会生成带“可沉淀原则 / 证据 / 时间线 / 仍需确认”的候选记忆，不只做普通归类。
- 向量近邻底座：新增本地稀疏向量索引通道，模糊问题会先找近邻候选；明确领域问题仍优先走领域召回，避免“还记得/上次”这类套话污染排序。
- 反馈校准：聊天透镜反馈不只改提及策略，还会调整 `memoryStrength`、`emotionalSalience` 和复查间隔。
- 可持久语义签名：`LongTermMemory` 新增 `semanticSignature` / `semanticSignatureVersion`，记忆创建、加载、合并和版本快照都会保留签名，向量索引用签名分桶但不硬过滤候选。
- Embedding 缓存底座：`AppState` 新增 `memoryEmbeddings`，本地状态迁移、保存和本机备份会刷新稳定缓存；回忆模式会把 embedding 缓存作为候选和排序加分；服务器新增 `/api/model/embeddings`，以后可接 OpenAI-compatible embedding 模型。
- 外部 embedding 查询向量：显式回忆旧事时，聊天发送链路会尝试在 3.5 秒内生成同模型查询向量和记忆向量；成功则参与本轮排序，失败则静默回落到本地索引。
- 高噪声抗干扰：评测集新增高权重泛化路线图干扰项，要求具体的长期遗忘焦虑仍然排在泛化项目记忆前面。
- 90% 门槛：`npm run test:memory` 当前为旧事召回 13/13，human-memory proxy gate 17/17。

## 未达标差距

- 还没有把外部 embedding 模型接入前端后台刷新；当前是本地稀疏向量索引 + 可持久语义签名 + 本地投影缓存，适合小窝初期，不适合海量记忆。
- 还没有真正的 LLM 后台反思整理；当前只有本地启发式 reflection 候选。
- 还没有跨设备云端增量同步冲突合并策略。
- 还缺更大规模真实聊天旧事评测集；当前评测覆盖核心机制，但样本量还小。
- 图片、文档、语音里的信息还不能自动进入记忆，必须先转成文本或链接。
- 情绪显著性仍是本地规则估算，不等于真正心理模型；后续要结合用户反馈和真实调用结果校准。
- 可解释反思仍是本地启发式，不是真正 LLM 深度自传式反思；后续要接入后台模型任务。

## 下一步

1. 加 embedding/语义检索，补齐更大规模“同义表达也能想起”。
2. 加后台反思整理任务，把多个事件提炼成稳定偏好或项目规则。
3. 扩充记忆评测脚本：加入更多真实聊天旧问题，检查召回到的 memoryIds 和回答依据。
4. 加云端增量同步和冲突合并，防止多端记忆分裂。
5. 接入文档/图片解析后，把来源证据保存到 memory sources。
