# Deep Research 记忆系统收尾记录

这份记录对应桌面上的 `deep-research-report.md`。姐姐采用了其中适合当前阶段的部分，也明确推迟了暂时会拖慢项目的部分。

## 已经采纳

- 作用域优先：每条记忆都有全局、关系、角色私有、项目、世界、分支、会话或临时空间。
- 候选审核：自动捕捉到的不确定关系记忆先进入待确认区，确认前不参与聊天提示。
- 冲突检查：先覆盖全局关系串戏、重复不一致、偏好相反这些高频问题。
- Prompt 打包：发送前按边界、稳定资料、关系、项目、事件分层注入，不全量塞入。
- 用户主权：编辑、删除、回收、恢复、永久删除、防复活 tombstone 都保留。
- 透明性：聊天页显示本轮记忆，记忆页显示最近调用，档案页显示来源、版本和调用记录。
- 提及时机：新增可自然提起、相关时使用、问起再提、只做边界四档策略。
- 冷却机制：新增 cooldown，避免刚纠正过或暂时敏感的记忆继续影响回复。

## 暂时不做

- 不急着上向量数据库。当前高价值记忆主要是结构化偏好、关系、项目和边界，先用可解释检索更稳。
- 不急着上后端云同步。现在目标是本机体验和架构稳定，云端会引入账号、同步冲突和隐私成本。
- 不急着做复杂多智能体记忆委员会。它听起来高级，但现在更需要把单人陪伴体验打磨顺。
- 不急着把聊天全文都 embedding。先保留来源、摘要和事件层，等数据量真的大起来再接 embedding adapter。

## 下一阶段再考虑

- IndexedDB 规范化多表，而不是单状态对象。
- 事件日志和 revision log 作为未来同步真源。
- Episode timeline，把“发生过什么”从稳定事实里拆出来。
- Scope Explorer，用树状方式查看角色、关系、世界、项目和继承关系。
- 可选 embedding adapter，只负责模糊召回，不负责判断事实真假。

## 当前收官判断

Sakura Pocket 的记忆系统已经从“记忆条目管理器”升级到“早期 memory runtime”：它不只保存内容，还开始决定谁能看见、什么时候能提、是否需要确认、是否正在冷却、用了哪些记忆、删掉后是否防复活。

这轮最重要的选择是克制：先把陪伴型记忆的边界、提及时机和用户主权做稳，再把向量、云同步、多端这些重能力往后接。
