# human-memory-90 硬任务

目标：Yuri Nest 的记忆系统向“人类记忆 90%+ 代理能力”推进。这里的 90% 不是宣称等于真人，而是 17 个工程代理维度里至少 16 个必须自动验证通过。

## 当前门槛

- `npm run test:memory` 必须通过。
- 旧事召回用例通过率必须 >= 90%。
- 17 维 human-memory proxy gate 必须 >= 90%。
- 静默/极敏记忆不能在回忆模式泄露。
- 明确旧事询问时，排序要优先响应问题线索，而不是被固定高权重规则压住。
- 召回一次后，相关记忆必须被巩固：`accessCount` 增加、`memoryStrength` 增加、`nextReviewAt` 更新。
- 后台整理必须能生成 reflection 候选。
- 相反偏好必须优先进入 value conflict。
- 人工修正记忆必须保留 revision history。
- 时间线线索必须能召回对应阶段事件。
- 情绪显著性必须让妹妹最焦虑、最在意、反复强调的事更难沉下去。
- 反思整理候选必须可解释：包含可沉淀原则、证据、时间线和仍需确认。
- 模糊旧事必须走本地向量近邻候选，但明确领域问题不能被向量噪声盖住。
- 每条记忆必须带可持久语义签名，为后续外部 embedding / ANN 向量库留稳定挂点。
- AppState 必须保存独立的 embedding 缓存，旧状态迁移、保存和备份时自动刷新。
- 外部 embedding 查询向量必须能进入回忆模式，且不可用时自动退回本地索引。
- 高权重泛化路线图不能压住更具体、更有情绪显著性的旧事焦虑。
- 聊天透镜反馈必须校准强度和显著性，不只是改展示策略。

## 17 维代理指标

| 维度 | 当前验证 |
|---|---|
| 语义召回 | 换说法也能召回维护性焦虑 |
| 程序/偏好记忆 | 召回相处规则和百合底线 |
| 项目事实记忆 | 召回模型接入、文档图片能力边界 |
| 事件时间线 | 召回五一假期阶段事件 |
| 关联回忆 | 通过同来源/同标签串起服务器与记忆同步提醒 |
| 隐私边界 | 回忆模式也不泄露 silent 记忆 |
| 复习加固 | 调用后增强强度并安排下次复查 |
| 整合与重巩固 | reflection 候选、冲突检测、版本线 |
| 时间线定位 | 识别 5月5号 / 五一最后一天 等阶段线索 |
| 情绪显著性 | 识别“焦虑/担心/放不下”的记忆优先级 |
| 可解释反思 | reflection 候选包含原则、证据、时间线、确认提示 |
| 向量近邻检索 | 模糊长周期遗忘问题能通过本地向量候选找回 |
| 可持久语义签名 | 记忆持久化保存 top semantic signature，索引用签名分桶但不硬过滤 |
| Embedding 缓存底座 | `memoryEmbeddings` 可复用稳定记录，并已接入回忆模式候选和排序 |
| 外部 Embedding 查询向量 | 回忆模式可使用同模型查询向量；失败或超时退回本地索引 |
| 高噪声抗干扰 | 高权重泛化项目/路线图记忆不能埋掉具体旧事 |
| 反馈校准 | 冷却/归档会降低强度和显著性，敏感反馈会加强保护 |
